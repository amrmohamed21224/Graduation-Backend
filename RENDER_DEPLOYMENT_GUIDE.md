# 🚀 دليل رفع مشروع التخرج (Node.js) على منصة Render من الصفر

متفهم جداً إحباطك من الانتظار، وأعتذر بشدة عن التأخير التقني الذي حدث سابقاً. هذا الدليل مكتوب خصيصاً لك لتفهم كل خطوة **(هنعمل إيه، وليه، وعشان إيه)** بأسلوب مبسط جداً لأنك ترفع مشروع Node لأول مرة.

## 📌 الفكرة العامة: كيف يعمل الـ Deployment؟
موقع **Render** (أو أي سيرفر سحابي) لا يأخذ الملفات من جهازك مباشرة عن طريق (Copy & Paste). السيرفر يحتاج إلى وسيط موثوق يسحب منه الكود، وهذا الوسيط هو موقع **GitHub**.
لذلك الدورة ستكون كالتالي: **جهازك (اللوكال) ⟵ نرفع الكود على GitHub ⟵ موقع Render يسحب الكود من GitHub ويقوم بتشغيله.**

---

### 🟢 الخطوة الأولى: التجهيز المحلي (في جهازك)
**الهدف:** التأكد من أن السيرفر السحابي (Render) لن يقوم بتحميل ملفات لا داعي لها (مثل فولدر `node_modules` الذي يستهلك مساحة ضخمة بلا فائدة).

1. **إعداد ملف `.gitignore`:**
   تأكد أن مشروعك `grad-project` يحتوي على ملف باسم `.gitignore` (يبدأ بنقطة). افتح الملف وتأكد من وجود هاتين الكلمتين داخله:
   ```text
   node_modules
   .env
   ```
   **ليه بنعمل كده؟** 
   - فولدر `node_modules` ضخم جداً، وكل سيرفر يحتاج لتنزيله بنفسه حسب بيئة التشغيل الخاصة به (عن طريق الأمر `npm install`). 
   - ملف `.env` لأنه يحتوي على الأرقام السرية الخاصة بايميلك والباسورد وقاعدة البيانات، ولا يجب أن يراها أي شخص يستعرض كود مشروعك على الـ GitHub.

---

### 🔵 الخطوة الثانية: تحزيم الكود ورفعه على GitHub
**الهدف:** نقل مجلد المشروع من جهازك إلى مستودع رقمي (Repo) على الإنترنت ليتمكن Render من قراءته.

1. افتح موقع **[GitHub](https://github.com/)** وقم بإنشاء حساب وتسجيل الدخول إذا لم تكن تمتلك واحداً.
2. انقر على علامة **(+)** في الشريط العلوي واختر **New repository**.
3. أطلق اسماً على المشروع (مثال: `watheq-backend`)، واجعله **Private** (خاص) أو **Public** (عام) واضغط **Create**.
4. سيُظهر لك جيت هاب شاشة سوداء بها عدة أوامر... افتح الآن شاشة الـ Terminal لديك بداخل فولدر المشروع `grad-project` واكتب الأوامر التالية بالترتيب (كل سطر ثم إنتر):
   ```bash
   git init
   git add .
   git commit -m "اكسر حاجز الخوف: أول تحديث للباك إند"
   git branch -M main
   # استبدل السطر التالي بالرابط الذي يظهر لك أنت في صفحة الجيت هاب الخاصة بك
   git remote add origin https://github.com/YourUsername/watheq-backend.git
   git push -u origin main
   ```
   **مبروك!** الآن كود الباك إند الخاص بك مرفوع ومحفوظ أونلاين بصيغة آمنة.

---

### 🟠 الخطوة الثالثة: الربط مع منصة Render
**الهدف:** إنشاء السيرفر الفعلي وربطه بالمستودع الذي رفعناه للتو.

1. افتح موقع **[Render.com](https://render.com/)** وسجل إما بحسابك الجديد في GitHub (Log in with GitHub - وهذا الأسهل).
2. من لوحة التحكم الرئيسية (Dashboard)، اضغط على الزر الكبير **New +** واختر **Web Service**.
3. ستفتح صفحة تعرض لك قائمة المستودعات الخاصة بك على GitHub. اختر المستودع الخاص بالباك إند `watheq-backend` واضغط **Connect**.

---

### ⚙️ الخطوة الرابعة: إعدادات السيرفر (مهمة جداً لنجاح التشغيل)
**الهدف:** إخبار سيرفرات Render بالكيفية والأوامر اللازمة لتشغيل مشروعك بنجاح. بمجرد ضغطك على Connect، ستفتح لك صفحة الـ Configuration. املأها كما يلي:

- **Name:** اسم السيرفر الذي سيظهر في الرابط، (مثلاً `watheq-api`).
- **Region:** اختر أقرب منطقة جغرافية لمصر لتقليل تأخير استجابة السيرفر (منطقة **Frankfurt** في أوروبا تعتبر سريعة وممتازة).
- **Branch:** اتركه `main`.
- **Environment:** تأكد أنه `Node`.
- **Build Command:** اكتب: `npm install` 
  *(تذكر المجلد المحذوف node_modules؟ هذا الأمر سيجعل Render يقوم بتنزيله وتركيبه داخل سيرفرهم فوراً).*
- **Start Command:** اكتب: `npm start`
  *(هذا الأمر الذي سيشغل السيرفر، وتذكر أننا وضعناه مسبقاً في ملف package.json لكي يعمل بدون مشاكل).*
- **Instance Type:** اختر الخطة المجانية **(Free - 0$/month)**.

---

### 🔑 الخطوة الخامسة: متغيرات البيئة السرية (Environment Variables)
**الهدف:** تغذية السيرفر بالأسرار والإعدادات التي أخفيناها في خطوة الـ `.gitignore` (متغيرات ملف الـ `.env`).

في نفس صفحة الإعدادات، انزل بالمقشر للأسفل واضغط على زر التحكم المسمى **Advanced**، ثم ابحث عن قسم **Environment Variables**.
اضغط على زر **Add Environment Variable** واستمر في إضافة كل متغير واحد تلو الآخر ليطابق تماماً ما هو موجود في ملف `.env` على جهازك:

| اسم المفتاح (Key) | القيمة (Value) | فائدته |
| :--- | :--- | :--- |
| `PORT` | `10000` | بورت التشغيل على سيرفر Render. (رقم 10000 آمن جداً للسيرفرات السحابية). |
| `MONGODB_URI` | `mongodb+srv://...` | يجب أن يكون رابط من لوحة تحكم MongoDB Atlas على النت وليس Localhost وإلا السيرفر لن يرى الداتا بيز الداخلي لجهازك! |
| `JWT_SECRET` | `أضف النص السري الخاص بك` | مفتاح التشفير وفك التشفير لحماية الـ API. |
| `JWT_EXPIRES_IN` | `90d` | مدة صلاحية توكن تسجيل الدخول. |
| `EMAIL_USER` | `ايميلك هنا` | من أجل نظام تنبيهات Nodemailer ليتمكن السيرفر من إرسال إيميلات حقيقية. |
| `EMAIL_PASS` | `الباسورد الخاص للتطبيقات` | يجب استخدام باسوورد تطبيقات Application Password من إعدادات جوجل وليس مالت الإيميل العادية. |
| `FRONTEND_URL` | `رابط الفرونت إند` | الرابط الخاص بصديقك (مطور الفرونت) لمنع أخطاء الـ CORS المزعجة عند الرفع. |

*(⚠️ تنبيه: لا تضع أي مسافات في الأسماء Key).*

---

### 🚀 الخطوة السادسة والأخيرة: إطلاق السيرفر (Deploy)
الآن وبعد أن قمت بإعداد كل شيء كالمحترفين، انزل لآخر الصفحة واضغط على الزر الأزرق الكبير: **Create Web Service**.

- سيأخذك لشاشة بها Terminal أسود يعرض سجل عملية البناء (Downloading modules... npm run start وغيرها).
- انتظر قرابة الدقيقتين حتى ترى رسالة باللون الأخضر بكلمة **Live 🎉**.
- في أعلى اليسار، تحت اسم السيرفر الخاص بك، سيظهر لك الرابط المباشر للباك إند الخاص بك (مثل `https://watheq-api.onrender.com`). 
- هذا هو الرابط الذهبي الذي ستقوم بنسخه وإعطائه لصديقك مطور الـ Frontend ليقوم هو بفك شفرة ה-URLs واستبدال كلمة `localhost:3000` عنده بهذا الرابط!

### ✅ مراجعة سريعة وأهم ميزة!
بهذا الشكل، الكود الخاص بك محمي، يعمل على الدوام 24 ساعة، ويتم تحديثه أوتوماتيكياً بطريقة مريحة (CI/CD).
**المعنى الدقيق:** في المستقبل، إذا قمت بتعديل أو تصليح الكود المتبقي في جهازك (اللابتوب)، لن تحتاج لفتح موقع Render أبداً... بمجرد أن تكتب أمر `git push` لرفع التعديل إلى جيت هاب، سيلحظ موقع Render التعديل فوراً، ويسحب التعديلات أوتوماتيكياً، ويعيد تشغيل السيرفر من تلقاء نفسه! ✨
